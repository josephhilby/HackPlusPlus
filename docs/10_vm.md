# 10 — Virtual Machine (VM) Specification

This document specifies the **Hack++ Virtual Machine (VM)** layer: a portable, stack-based runtime abstraction that sits **above the ISA** and **below the compiler**.

The VM provides a machine-independent execution model—**stack evaluation**, **segmented memory**, and **structured control flow**—that can be deterministically lowered into Hack++ assembly.

**Related:**

* [System Integration](./08_system.md)
* [Instruction Set Architecture](./09_instruction_set.md)
* [Memory Hierarchy](./06_memory.md)

---

## Design Notes

**Runtime vs. compile-time abstractions**

* **ISA / Assembly** is a *linear*, instruction-by-instruction execution interface (machine-facing).
* **VM** is a *runtime abstraction* that encodes higher-level intent (stack operations, function calls) while remaining deterministic.
* The compiler targets the VM (or ISA directly), and the VM translator lowers VM → ASM → machine code.

**Single operand stack**
All arithmetic and logical operations consume and produce values via a single **data stack**.

**Segmented memory model**
The VM exposes **logical segments** (local/argument/this/that/temp/static/pointer/constant) that map onto concrete RAM addresses at runtime.

---

## Data Types

The VM operates on a single word type:

* **int**: 16-bit signed integer (two’s complement)
* **bool**: encoded as a VM int

    * `true` = `-1` (all bits set: `0xFFFF`)
    * `false` = `0`

> Note: Using `-1` for true makes boolean logic (`and`, `or`, `not`) line up with bitwise operations.

---

## Memory Segments

The VM exposes **eight** segments.

| Segment    | Purpose                  | Allocation / Behavior                                 |
| ---------- | ------------------------ | ----------------------------------------------------- |
| `local`    | Callee local variables   | Base = `LCL`, sized by `nlocals` at function entry    |
| `argument` | Caller-passed arguments  | Base = `ARG`, sized by `nargs` at call setup          |
| `this`     | Heap/data-structure base | Base = `THIS` (typically points into heap)            |
| `that`     | Heap/data-structure base | Base = `THAT` (typically points into heap)            |
| `temp`     | Scratch registers        | Fixed addresses `RAM[5..12]`                          |
| `static`   | File-scoped statics      | Fixed range `RAM[16..255]`, addressed as `FileName.i` |
| `pointer`  | Selects THIS/THAT bases  | `pointer 0` ↔ `THIS`, `pointer 1` ↔ `THAT`            |
| `constant` | Immediate literals       | Pseudo-segment; values are not stored in RAM          |

> Canonical RAM symbol bindings (`SP`, `LCL`, `ARG`, `THIS`, `THAT`, `R13..R15`) are defined in the ISA and memory docs.

## VM Grammar (EBNF)

### Tokens

```regexp
integer    := ^[0-9]+$
identifier := ^[A-Za-z_$:.][A-Za-z0-9_-]*$
newline    := [\r\n]
```

### Syntax

```ebnf
non-terminal  ::= production rule
---               ---
program       ::= { line }

line          ::= [ command ] [ comment ] newline
comment       ::= "//" { any_char_except_newline }

command       ::= arithmetic | memory | branching | function

arithmetic    ::= "add" | "sub" | "neg" | "eq" | "gt" | "lt" | "and" | "or" | "not"

memory        ::= ("push" | "pop") segment index  (* segment[index] *)
segment       ::= "local" | "argument" | "this" | "that" | "temp" | "static" | "pointer" | "constant"
index         ::= integer

branching     ::= ("label" | "goto" | "if-goto") label

function      ::= "function" function_name nlocals
               |  "call" function_name nargs
               |  "return"

label         ::= identifier
function_name ::= identifier
nlocals       ::= integer  (* number of locals allocated by callee *)
nargs         ::= integer  (* number of args passed by caller *)
```

### Legend

* `{ … }` = zero or more
* `[ … ]` = optional (zero or one)
* `|` = alternative
* Mnemonics and identifiers are **case-sensitive**

## Execution Semantics

This section defines the *observable effects* of VM commands in terms of stack and memory state.

### Stack Model

* The stack grows upward in RAM.
* `SP` points **one past** the top element.

Definitions:

* `push(v)` means: `RAM[SP] = v; SP = SP + 1`
* `pop()` means: `SP = SP - 1; v = RAM[SP]`

### Arithmetic / Logical Commands

All arithmetic commands operate on stack operands.

| Command | Stack Effect   | Meaning                          |
| ------: | -------------- | -------------------------------- |
|   `add` | `x y → (x+y)`  | 16-bit two’s complement addition |
|   `sub` | `x y → (x−y)`  | subtract top from next           |
|   `neg` | `x → (−x)`     | arithmetic negation              |
|   `and` | `x y → (x&y)`  | bitwise AND                      |
|    `or` | `x y → (x\|y)` | bitwise OR                       |
|   `not` | `x → (~x)`     | bitwise NOT                      |
|    `eq` | `x y → bool`   | true iff `x == y`                |
|    `gt` | `x y → bool`   | true iff `x > y` (signed)        |
|    `lt` | `x y → bool`   | true iff `x < y` (signed)        |

**Comparison results**

* true = `-1`
* false = `0`

### Memory Access Commands

|          Command | Effect                                                         |
| ---------------: | -------------------------------------------------------------- |
| `push segment i` | reads `segment[i]` (or immediate for `constant`) and pushes it |
|  `pop segment i` | pops value and writes it into `segment[i]`                     |

**Segment addressing rules (high level)**

* `local i` → `RAM[LCL + i]`
* `argument i` → `RAM[ARG + i]`
* `this i` → `RAM[THIS + i]`
* `that i` → `RAM[THAT + i]`
* `temp i` → fixed `RAM[5 + i]` (valid `i=0..7`)
* `pointer 0` → `THIS`, `pointer 1` → `THAT`
* `static i` → `RAM[16 + static_index(FileName, i)]` (file-scoped)
* `constant i` → immediate value `i` (no RAM read)

> The exact static symbol naming scheme is typically `FileName.i` at the assembly level.

### Branching Commands

|     Command | Effect                                                            |
| ----------: | ----------------------------------------------------------------- |
|   `label L` | declares a symbolic address `L` within the current function scope |
|    `goto L` | unconditional jump to `L`                                         |
| `if-goto L` | pops `x`; if `x != 0` then jump to `L`                            |

**Scope rule**
Labels are function-scoped. A conventional lowering scheme is `FunctionName$Label` to avoid collisions.

### Function Commands

Function commands define call/return semantics and stack frame management.

|              Command | Effect                                                                 |
| -------------------: | ---------------------------------------------------------------------- |
| `function F nlocals` | declares entry point `F`; allocates `nlocals` locals initialized to 0  |
|       `call F nargs` | pushes a call frame, repositions `ARG`/`LCL`, transfers control to `F` |
|             `return` | restores caller frame, places return value for caller, jumps back      |

**Canonical call frame (nand2tetris-compatible)**
A typical lowering uses the following saved state on the stack:

* return address
* saved `LCL`
* saved `ARG`
* saved `THIS`
* saved `THAT`

Then:

* `ARG = SP - (nargs + 5)`
* `LCL = SP`
* `goto F`

On `return`:

* `frame = LCL`
* `ret = RAM[frame - 5]`
* `*ARG = pop()` (place return value)
* `SP = ARG + 1`
* restore `THAT/THIS/ARG/LCL` from `frame - 1..4`
* `goto ret`

> This document specifies the *semantic contract*. The precise implementation details are owned by the VM translator.

---

## Notes and Constraints

* VM integers are **16-bit**; overflow wraps naturally in two’s complement.
* `temp` index must be `0..7`.
* `pointer` index must be `0..1`.
* `constant` values are typically restricted to `0..32767` in Hack toolchains.
